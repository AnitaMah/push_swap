Push_swap 42 Guide / Гайд Push_swap 42

Concept / Концепт

EN: You have two stacks (A and B). You must sort A using minimal operations.
UKR: У тебе є два стеки (A і B). Потрібно відсортувати A мінімальною кількістю операцій.

Disorder metric (Inversion ratio) / Метрика безладу (інверсії)

EN: This function measures how unsorted the stack is by counting inversions (pairs that are in wrong order). It helps estimate how far the array is from being sorted.
UKR: Ця функція вимірює, наскільки стек не відсортований, рахуючи інверсії (пари елементів у неправильному порядку). Вона показує, наскільки масив далекий від сортування.

Function idea / Ідея функції

function compute_disorder(stack a):
    mistakes = 0
    total_pairs = 0
    for i from 0 to size(a)-1:
        for j from i+1 to size(a)-1:
            total_pairs += 1
            if a[i] > a[j]:
                mistakes += 1
    return mistakes / total_pairs

What it means / Що це означає

EN: Each pair (i, j) is checked. If earlier element is bigger than later → inversion. Result is a ratio: 0 = sorted, 1 = fully reversed.
UKR: Кожна пара (i, j) перевіряється. Якщо лівий елемент більший за правий → це інверсія. Результат — це частка: 0 = відсортовано, 1 = повністю навпаки.

Example / Приклад

A = [1, 2, 3, 4]
mistakes = 0 → disorder = 0.0

A = [4, 3, 2, 1]
all pairs are wrong → disorder = 1.0

A = [1, 3, 2, 4]
one inversion (3 > 2) → small disorder

0. Preparation → Quicksort (Index Mapping)

Idea / Ідея

EN: Before push_swap starts, we need to convert values into indexes (0..n-1). For this we sort a COPY of the array using QuickSort.
UKR: Перед запуском push_swap ми перетворюємо числа в індекси (0..n-1). Для цього сортуємо КОПІЮ масиву через QuickSort.

Why QuickSort / Чому QuickSort

EN: QuickSort is fast in practice (O(n log n) average), in-place, and simple to implement in C.
UKR: QuickSort швидкий на практиці (O(n log n) в середньому), працює in-place і легко реалізується в C.

How QuickSort works / Як працює QuickSort

EN: QuickSort is a divide-and-conquer algorithm. It selects a pivot element, then splits the array into two parts: - elements smaller than pivot - elements greater than pivot Then it recursively sorts both parts.
UKR: QuickSort — це алгоритм "розділяй і володарюй". Він обирає pivot (опорний елемент) і ділить масив на дві частини: - менші за pivot - більші за pivot Потім рекурсивно сортує обидві частини.

Partition process / Процес розбиття

Array: [40, 10, 30, 20]
Pivot: 20

Step 1: compare each element
- 10 < 20 → left side
- 40 > 20 → right side
- 30 > 20 → right side

Result after partition:
[10, 20, 30, 40]
       ↑ pivot placed correctly

Step-by-step recursion idea / Рекурсія

quicksort(arr):
    if size <= 1: return

    pivot = choose_last_element
    left = elements < pivot
    right = elements > pivot

    quicksort(left)
    quicksort(right)

    merge: left + pivot + right

Key intuition / Головна інтуїція

EN: Each partition places at least one element in its final correct position.
UKR: Кожне розбиття ставить хоча б один елемент у його фінальну правильну позицію.

How it is used / Як використовується

EN: Yes — this step means you COPY the stack A into a temporary array (or dynamic array). You do NOT modify the stack. Then you sort this copy to build the index mapping.
UKR: Так — цей крок означає, що ти КОПІЮЄШ стек A у тимчасовий масив (або динамічний масив). Ти НЕ змінюєш стек. Потім ти сортуєш цю копію, щоб створити mapping індексів. 

Stack A (original):
[40, 10, 30, 20]

Step 1: copy to array
copy = [40, 10, 30, 20]

Step 2: sort copy
copy = [10, 20, 30, 40]

Step 3: assign indexes
10 -> 0
20 -> 1
30 -> 2
40 -> 3

Step 4: replace original values with indexes
A becomes:
[3, 0, 2, 1]

Step-by-step example / Приклад

Original: [40, 10, 30, 20]
Sorted:   [10, 20, 30, 40]

Mapping:
10 -> 0
20 -> 1
30 -> 2
40 -> 3

Result:
[3, 0, 2, 1]

Quicksort pseudocode

function quicksort(arr, low, high):
    if low < high:
        pivot = partition(arr, low, high)
        quicksort(arr, low, pivot - 1)
        quicksort(arr, pivot + 1, high)

function partition(arr, low, high):
    pivot = arr[high]
    i = low - 1

    for j from low to high - 1:
        if arr[j] <= pivot:
            i++
            swap(arr[i], arr[j])

    swap(arr[i+1], arr[high])
    return i + 1

C-style implementation idea

// copy array

    index[original[i]] = position_in_sorted_copy

Complexity / Складність

EN: Average O(n log n), worst O(n^2) (rare with good pivot strategy).
UKR: Середня O(n log n), гірша O(n^2) (рідко при нормальному pivot).

1. Simple → Selection Sort Adaptation

Idea / Ідея

EN: This method repeatedly extracts the minimum element from stack A and pushes it to stack B. It behaves like Selection Sort but adapted to stack operations (rotate + push).
UKR: Цей метод постійно знаходить мінімальний елемент у стеку A і переносить його в B. Це як Selection Sort, але адаптований під стекові операції (rotate + push).

How it works internally / Як це працює

EN: You cannot access elements randomly, so you simulate scanning using rotations. You bring the minimum element to the top using ra or rra, then push it.
UKR: Ти не можеш звертатися до елементів напряму, тому імітуєш пошук через обертання. Ти піднімаєш мінімум наверх через ra або rra, потім робиш push.

Key optimization / Головна оптимізація

EN: Always choose the shortest rotation path: - ra (rotate up) - rra (rotate down) This reduces operation count significantly.
UKR: Завжди обирай найкоротший шлях обертання: - ra (вгору) - rra (вниз) Це сильно зменшує кількість операцій.

Step-by-step example / Покроковий приклад

Initial:
A: [4, 2, 5, 1]
B: []

Step 1: min = 1
Best path: rra (faster than ra)
A → [1, 4, 2, 5]
pb
A: [4, 2, 5]
B: [1]

Step 2: min = 2
A: [4, 2, 5]
rotate → [2, 5, 4]
pb
A: [5, 4]
B: [2, 1]

Step 3:
A: [5, 4]
pb 4 then 5

Complexity / Складність

EN: O(n²) because each step requires scanning + rotating.
UKR: O(n²), тому що кожен раз ти шукаєш мінімум через повний прохід.

Why it is used / Чому його використовують

EN: It is simple, easy to implement, and good for very small inputs (≤ 5 or ≤ 10).
UKR: Він простий і ідеальний для малих наборів (≤ 5 або ≤ 10).

Pseudocode

while A not empty:
    min_pos = find_min_position(A)

    if min_pos < size(A)/2:
        while min not on top:
            ra
    else:
        while min not on top:
            rra

    pb

while B not empty:
    pa

Visualization / Візуалізація

A: 4 → 2 → 5 → 1
        ↓ find min

Rotate:
A: 1 → 4 → 2 → 5
    ↑ push

2. Medium → Chunk Sort

Idea / Ідея

EN: Chunk sort does NOT push elements in fully sorted order. Instead, it pushes numbers in small VALUE RANGES (chunks) from A to B. B becomes partially ordered, not sorted.
UKR: Chunk sort НЕ відправляє елементи одразу в повністю відсортованому вигляді. Воно працює по ДІАПАЗОНАХ значень (чанках). Стек B не є повністю відсортованим, а лише частково впорядкованим.

Why INDEX is important / Чому важливий INDEX

EN: In push_swap you never work with raw values directly. You first convert values into INDEXES (0..n-1). This is called normalization.
UKR: У push_swap ти не працюєш напряму з числами. Спочатку ти перетворюєш їх у ІНДЕКСИ (0..n-1). Це називається нормалізація. 

Example:
A = [40, 10, 30, 20]
Sorted = [10, 20, 30, 40]

Index mapping:
10 -> 0
20 -> 1
30 -> 2
40 -> 3

Now A becomes:
[3, 0, 2, 1]

Chunk definition uses INDEX / Чанки працюють через INDEX

EN: Chunks are NOT based on values, but on index ranges. For example chunk size = 2 means: [0-1], [2-3], [4-5]...
UKR: Чанки будуються НЕ по значеннях, а по ІНДЕКСАХ. Наприклад chunk size = 2 означає: [0-1], [2-3], [4-5]...

How it works internally / Як це працює

EN: You scan stack A and push only elements whose INDEX belongs to current chunk range. You rotate A (ra/rra) to reach needed elements faster.
UKR: Ти проходиш стек A і пушиш тільки ті елементи, чий ІНДЕКС входить у поточний чанк. Інші елементи ти крутиш (ra/rra), щоб швидше дістатися потрібних.

Important correction / Важливе виправлення

EN: B is NOT sorted during chunk phase. Sorting happens later when you push back largest elements from B to A.
UKR: B НЕ відсортований під час chunk фази. Сортування відбувається пізніше, коли ти повертаєш найбільші елементи з B назад в A.

Step-by-step example / Приклад

Original:
A = [40, 10, 30, 20]

After normalization:
A = [3, 0, 2, 1]

chunk size = 2
Chunks:
[0-1] -> small values
[2-3] -> big values

Process chunk [0-1]:
push 0 → B
push 1 → B

A: [3, 2]
B: [1, 0]

How reconstruction works / Як повертається сортування

while B not empty:
    find max INDEX in B
    bring max to top (rb / rrb)
    pa → move to A

Result A becomes sorted

Pseudocode

normalize values -> indexes
chunk_size = sqrt(n)
start = 0
end = chunk_size

while A not empty:
    if index(A.top) in [start..end]:
        pb
    else:
        ra

    if chunk finished:
        start += chunk_size
        end += chunk_size

# rebuild sorted A
while B not empty:
    bring max INDEX(B) to top
    pa

Visualization / Візуалізація

Values:    40 10 30 20
Indexes:    3  0  2  1

Chunk [0-1]:
B: 1 0

3. Complex → Radix Sort

Idea / Ідея

EN: Radix sort in push_swap works on INDEXES, not original values. After normalization, every number becomes 0..n-1, and we sort them using binary bits.
UKR: Radix sort у push_swap працює з ІНДЕКСАМИ, а не з оригінальними числами. Після нормалізації кожне число стає 0..n-1 і сортується через біти.

Why index is important / Чому важливий індекс

EN: Using indexes makes numbers small and dense, which reduces number of bits needed for sorting.
UKR: Індекси роблять числа маленькими і послідовними, що зменшує кількість бітів для сортування.

Binary mapping idea / Ідея бітового представлення

EN: Each index is converted into binary form. Radix processes one bit at a time from LSB → MSB (right to left).
UKR: Кожен індекс переводиться у двійкову форму. Radix обробляє по одному біту від молодшого до старшого. 

Example (indexes):
A = [3, 0, 2, 1]

Binary:
3 = 011
0 = 000
2 = 010
1 = 001

How bit decision works / Як працює вибір по біту

EN: For each bit position: - if bit == 0 → push to stack B (pb) - if bit == 1 → rotate A (ra) This groups numbers by binary significance.
UKR: Для кожної бітової позиції: - якщо біт == 0 → відправляємо в B (pb) - якщо біт == 1 → крутимо A (ra) Це групує числа по двійкових властивостях.

Step-by-step example / Приклад

A (indexes): [3, 0, 2, 1]

BIT 0 (LSB):
3 (1) → ra
0 (0) → pb
2 (0) → pb
1 (1) → ra

A after pass: [3, 1]
B: [0, 2]

Move back:
pa pa → A partially sorted

Bit table visualization / Таблиця бітів

EN: This table shows how each INDEX is decomposed into binary bits. Radix uses (index >> i) & 1 to read each bit.
UKR: Ця таблиця показує, як кожен ІНДЕКС розкладається на біти. Radix використовує (index >> i) & 1 для читання кожного біта. 

Index | Binary | Bit2 | Bit1 | Bit0
------------------------------------
0     | 000    |  0   |  0   |  0
1     | 001    |  0   |  0   |  1
2     | 010    |  0   |  1   |  0
3     | 011    |  0   |  1   |  1
4     | 100    |  1   |  0   |  0
5     | 101    |  1   |  0   |  1
6     | 110    |  1   |  1   |  0
7     | 111    |  1   |  1   |  1

EN: Reading rule: - Bit0 = (index >> 0) & 1 - Bit1 = (index >> 1) & 1 - Bit2 = (index >> 2) & 1
UKR: Правило читання: - Bit0 = (index >> 0) & 1 - Bit1 = (index >> 1) & 1 - Bit2 = (index >> 2) & 1

Full process / Повний процес

max_bits = number of bits in max index

for i in range(max_bits):
    for each element in A:
        if (index >> i) & 1 == 0:
            pb
        else:
            ra

    while B not empty:
        pa

Key insight / Головна ідея

EN: Radix works because indexes are already sorted domain (0..n-1), so binary grouping converges quickly.
UKR: Radix працює швидко, бо індекси вже впорядковані (0..n-1), і бінарне групування швидко приводить до сортування.

4. Adaptive Wrapper

Idea / Ідея

EN: Choose algorithm based on BOTH size of the stack and disorder level (computed by inversion ratio). This makes the solution adaptive, not static.
UKR: Обираєш алгоритм залежно як від розміру масиву, так і від рівня безладу (інверсій). Це робить рішення адаптивним, а не фіксованим.

Disorder-aware decision / Вибір з урахуванням безладу

EN: Different disorder levels require different algorithmic complexity.
UKR: Різний рівень безладу потребує різної алгоритмічної складності.

Low disorder / Малий безлад

if disorder < 0.2:
    use O(n²)
    → selection sort adaptation

EN: Stack is almost sorted, so simple algorithms are still efficient.
UKR: Стек майже відсортований, тому прості алгоритми ще ефективні.

Medium disorder / Середній безлад

if 0.2 ≤ disorder < 0.5:
    use O(n√n)
    → chunk sort

EN: Stack is partially shuffled. Chunk grouping reduces unnecessary movements.
UKR: Стек частково перемішаний. Chunk grouping зменшує зайві рухи.

High disorder / Великий безлад

if disorder ≥ 0.5:
    use O(n log n)
    → radix sort

EN: Stack is highly chaotic. Radix works efficiently regardless of disorder.
UKR: Стек дуже хаотичний. Radix ефективно працює незалежно від рівня безладу.

Adaptive wrapper pseudocode

normalize_indexes(A)

entropy = compute_disorder(A)

if size(A) <= 5:
    selection_sort(A)

else if entropy < 0.2:
    selection_sort(A)

else if entropy < 0.5:
    chunk_sort(A)

else:
    radix_sort(A)

Why disorder matters / Чому важливий безлад

EN: Two arrays with the same size can require different strategies depending on how shuffled they are.
UKR: Два масиви однакового розміру можуть потребувати різних алгоритмів залежно від того, наскільки вони перемішані.

Compute disorder reminder / Нагадаємо функцію

function compute_disorder(stack a):
    mistakes = 0
    total_pairs = 0
    for i from 0 to size(a)-1:
        for j from i+1 to size(a)-1:
            total_pairs += 1
            if a[i] > a[j]:
                mistakes += 1
    return mistakes / total_pairs

Common 42 mistakes / Типові помилки (push_swap)

sa, sb, ss → swap
ra, rb, rr → rotate
rra, rrb, rrr → reverse rotate
pa, pb → push

What do these operations mean / Що означають ці операції

EN: These are the ONLY allowed operations in push_swap. They modify stacks A and B.
UKR: Це ЄДИНІ дозволені операції в push_swap. Вони змінюють стеки A і B.

  • sa / sb / ss (swap)
    EN: Swap the first two elements on stack A or B (or both for ss).
    UKR: Міняє місцями перші два елементи у стеку A або B (ss — в обох).
  • ra / rb / rr (rotate)
    EN: Move the first element to the bottom of the stack (shift up).
    UKR: Переносить перший елемент у кінець стеку (зсув вгору).
  • rra / rrb / rrr (reverse rotate)
    EN: Move the last element to the top of the stack (shift down).
    UKR: Переносить останній елемент у початок стеку (зсув вниз).
  • pa / pb (push)
    EN: Move the top element from one stack to the other. - pa: B → A - pb: A → B
    UKR: Переносить верхній елемент з одного стеку в інший. - pa: з B в A - pb: з A в B
